# 使用问题 FAQ

## 常用命令有哪些？

对话中的命令（斜杠开头）：
- `/clear` — 清空当前对话，重新开始。场景：聊歪了想重来
- `/rewind` — 回滚上一步操作（或按两次 ESC）。场景：AI 改错了代码想撤回
- `/init` — 项目初始化，自动生成 CLAUDE.md。场景：开始新项目时
- `/cost` — 查看当前对话花了多少 token/钱。场景：想控制成本时
- `/compact` — 压缩对话历史，减少 token 消耗。场景：对话太长快到上限时
- `/terminal-setup` — 设置终端快捷键（装完建议先跑一次）

启动时的参数：
- `claude` — 正常启动
- `claude --dangerously-skip-permissions` — 全自动模式（AI 可以自己执行所有操作，不再逐个询问你）
- `claude --resume` — 恢复上次对话

对话中的快捷操作：
- `@` + 文件名 — 引用文件给 AI 看
- 拖文件到终端 — 也能引用文件
- Alt+Enter — 换行（Windows），Option+Enter（Mac）
- ESC×2 — 停止 AI 当前操作

## CLAUDE.md 是什么？有什么用？

它是给 AI 看的「说明书」或「记忆文件」。每次启动 Claude Code，它会自动读取。

你可以写进去：项目背景、你的偏好、常用规范、上次进度。

因为 AI 没有长期记忆，CLAUDE.md 就是它的「外挂记忆」。关闭项目前让 AI 把需要记住的东西写进去，下次就能接着上次继续。

## 怎么让 AI 全自动执行？

启动 Claude Code 时加参数：
```
claude --dangerously-skip-permissions
```

这个模式下 AI 可以自动执行所有操作（创建文件、修改代码、运行命令等），不会一个个问你「要不要执行」。

什么时候用：
- 你已经明确告诉 AI 要做什么，不想每步都点确认
- 批量处理任务时（比如让 AI 自动整理 100 个文件）
- 你信任 AI 的判断，想让它一口气干完

什么时候别用：
- 第一次做某类任务时，先用普通模式看看 AI 怎么做
- 涉及重要文件/数据时，还是一步步确认更安全

如果 AI 跑偏了：
- 按两次 ESC 立刻停止
- 用 `/rewind` 回滚到上一步
- 用 Ctrl+C 强制退出

## 怎么引用文件给 AI 看？

三种方式，都很方便：
1. `@`引用：在对话框输入 `@` 然后打文件名，会自动补全，选中回车
2. 拖拽：直接把文件从文件夹拖到终端窗口
3. 说路径：直接告诉 AI「帮我看一下 D:/项目/报告.docx」

支持的文件类型：
- 代码文件（.js/.py/.html 等）✅
- Markdown 文件（.md）✅
- Word 文档（.docx）✅
- Excel 表格（.xlsx）✅
- PDF 文件 ✅
- 图片（.png/.jpg）✅ Claude 能看图
- PPT（.pptx）✅ 让 AI 试试就行

如果 AI 说读不了某个格式，告诉它「你试试看，应该能读的」，很多时候它就能读了。

## Claude Code 感觉比网页版还难用，只能识别 MD 文档？

定位不同：网页版是通用对话助手，Claude Code 是命令行工具，能力边界更大。

关于「只能识别 MD」是误解，它几乎可以读所有文件，包括图片和 PDF。图片拖进去显示路径文字，但其实 AI 已经看到了。

真正优越性：直接读写电脑文件、执行命令、创建智能体、联通飞书等工具。

建议：听 Claude Code 课程，回看直播答疑，多尝试！

## AI 说只能读 MD 文件，其他格式读不了？

这是可以突破的。有时 AI 说不能读某格式，让它自己尝试：「你尝试一下读这个 PPTX，怎么会读不了？你试试看。」

这些格式都能读：PDF、图片、Excel、PPT、Word。

小技巧：AI 说读不了，就让它自己想办法解决。

## 我的工作文字数据处理极少，AI 能帮什么？

思维转换：不要把「文字」理解成「文本文件」，而是「通往终点的语言和梳理过程」。

AI 帮你：沉淀设计逻辑、梳理场景规范、做思路梳理师。

机械工程师也适用：图纸审核清单、参数标准、工艺 SOP。

建议：直接问 AI「我是 XX 岗位，日常工作是 XX，你觉得能怎么帮我搭建智能体？」

## Claude Code 聊天记录第二天就没了怎么办？

Claude Code 本身没有聊天记录，这是正常的。但有 CLAUDE.md 等记忆文档。

做法：
1. 结束前说「看看今天有什么需要记忆的吗？明天要继续做 XXX」
2. 第二天说「继续 XXX」或「昨天聊的 XXX 你还记得吗」

本质：建立「记忆约定」，每天结束存档，第二天读档继续。

## 多行输入怎么换行？

先运行 `/terminal-setup`，然后用 Alt+Enter（Windows）或 Option+Enter（Mac）换行。

## AI 给出 1、2、3 几个选项，只能选一个，没法补充？

最后一个选项是 Other（其他），可以自己输入。

如果选了 Other 忘了选项 1 内容：
1. 先选 1 让它跑，中途可以打断补充
2. 或说「把刚才选项 1 再说一遍」

## 对话太长达到上限，内容没保存就丢失了？

方法一：手动复制保存，从下往上拉，新建 md 文件保存，新开终端继续。
方法二：`/rewind` 回溯到之前节点。

预防：不要一次性丢太多资料，分轮慢慢给，AI 会自己压缩。

## Claude Code 一直报 Error writing file 怎么办？

三步解决：
1. 按 ESC 让它停下来
2. 停了之后说「继续」
3. 还不行就开新终端，复制重要内容过去继续

本质：会话状态卡住了，不是代码问题。果断开新会话最高效。

## 如何快速将 MD 文档转换成 Word、Excel、PPT 等格式？

- MD→Word：直接复制粘贴，或让 Claude Code 生成
- MD→Excel：Claude Code 直接生成，或走飞书多维表格导出
- MD→PPT：Claude Code 能生成但样式朴素，想好看用 Kimi/Gamma

进阶：可以让 AI 一次性生成多个不同格式文件。

## CLAUDE.md 文档需要手动建立吗？下一个项目要复制过去吗？

不需要手动建立，Claude Code 会自动创建。

怎么用：每次跟 AI 说「请帮我记录 XXX」，它会自动记录。

进阶：大项目下的小项目可以建分开的 memory 文件夹。

跨项目搬运：让 AI 自己搬，说「帮我把这个东西搬到另一个项目」。

## AI 可以画业务流程图吗？

可以！
- 方式一：直接让 Claude 画，用文字/符号画简易版
- 方式二：让 Claude Code 生成 HTML 可视化流程图（推荐）

关键：先在聊天里梳理清楚流程，最后才画图。

## 如何让 Claude Code 同时运行多任务？

- 方法一：拆 Project（多文件夹），每个文件夹开一个 Claude Code
- 方法二：拆 Terminal（多终端），同一个项目开多个 Terminal 同时跑

最后汇总方式随意，MD 汇总或直接对话都行。

## 梳理完工作流后 AI 给了我一个指令，这就是工作流了吗？是真正的 Agent 吗？

工作流概念比较模糊。目前技术水平能实现：让 AI 有手有脚（读写文档、操作文件），打通权限，建网站、发通知、自动执行。完整的工作流还需要时间慢慢完善。

## 是不是每次对话后都要说「请创建 readme.md」？

可以这样做，但也可以更灵活。和 AI 共创「我们需要记住哪些东西」，它会帮你拆分：背景信息类、底层内核类、工作步骤类。怎么命名分类，在使用中慢慢优化。

## 文件夹从桌面移动到 D 盘，Claude 是不是就找不到了？

不会，告诉它新位置就能读。它还能帮你搬运文件，说「帮我把这个文件从 D 盘搬到 E 盘」就行。

## Claude 网页版提示 too many chats going，删了聊天也不行？

这个报错大概率不是指左边栏的聊天数量，而是浏览器同时打开了太多个 Claude 的标签页。

解决方法：
1. 关掉所有 Claude 的浏览器标签页，只留一个
2. 如果还不行，清一下浏览器缓存，或者换个浏览器试试
3. 还不行的话，退出登录重新登一次

建议：尽量在一个大的 Project 下面开 chat，而不是单独开很多独立的 chats。

## 如何省 token？切换 Plan Mode 能省吗？

Plan Mode 不省 token，但增加稳定性——防止 AI 想完就自己执行，执行错了还把旧版本改坏。

省 token 的方法主要三个：
1. 自己的逻辑清晰 + 表达收敛：这是最有效的省 token 方式。说得越清楚，AI 走的弯路越少。
2. 使用经验积累：用得越熟越知道怎么高效调用 AI，前期多花点钱是正常的。
3. 让 AI 记住高效路径：某个流程跑通了，告诉 AI「记住这个最短路径，以后直接这样做」，下次就省了。

## OpenClaw（龙虾）是什么？和 Claude Code 有什么区别？

龙虾（OpenClaw）是一个 AI Agent 框架，给 Claude、Kimi 等大模型套上了一套「有手有脚、有记忆、有理解力」的执行机制。

龙虾 vs Claude Code 5 个核心区别：
1. 界面更友好：用户 base 而非程序员 base，普通用户也能上手，不需要懂代码
2. 便携性强：手机就能用，不需要守在电脑前
3. 协作性好：可以拉进群，团队成员都可以在群里调用它背后的知识库和信息表格
4. Sub-Agent：任务可拆分并行跑，效率大幅提升
5. 主动机制：龙小会主动来找你，可以配置主动巡检、定时提醒

新手建议：如果现在还处于课程初期，可以先把 Claude Code 用好，未来一键迁移非常方便。

## CLAUDE.md 和记忆文件有什么区别？怎么做好存档？

CLAUDE.md 是你写给 AI 的「项目说明书」，记忆文件是 AI 自己的「笔记本」。

- CLAUDE.md：放在项目根目录，每次 Claude 进入项目都会先读。你自己写、自己维护。
- 记忆文件（.claude/memory/）：AI 在对话中自己记下来的经验和发现。主要让 AI 自己维护。

存档经验：
1. 分层级存档：用户级（所有项目通用）vs 项目级（项目专属）
2. 框架与目标文档：做事前先讨论框架、逻辑、目标，形成 MD
3. 信息/方法论背景：给 AI 必要的背景资料，形成 MD
4. 关键过程文档：评估选型、项目优先级、进度追踪等
5. 具体实践文档：具体执行的存档

## CLAUDE.md 和 Skills 有什么区别？

打个比方：CLAUDE.md 就像贴在工位上的「工作须知」，AI 每次干活都会先看一遍；Skills 就是抽屉里的「操作手册」，需要用的时候才拿出来。

四个维度对比：
1. 加载方式：CLAUDE.md 每次对话自动加载；Skills 需要触发才调用
2. 作用：CLAUDE.md 告诉 AI 你是谁、怎么做事；Skills 定义具体任务的标准操作流程
3. 内容：CLAUDE.md 放背景和规则；Skills 放任务的详细步骤
4. 触发：CLAUDE.md 永远生效；Skills 用 `/技能名` 或关键词触发

## CLAUDE.md 和记忆约定是什么关系？用户级和项目级分别存什么？

架构图：

用户级：`C:\Users\用户名\.claude\`
- `CLAUDE.md` — 全局记忆，打开任何文件夹都能读到
- `commands\` — 全局 Skills（任何项目都能触发）

项目级：每个项目文件夹下都有自己的 CLAUDE.md 和 `.claude\commands\`

加载顺序：用户级 CLAUDE.md → 项目级 CLAUDE.md（两层叠加，不冲突）

用户级存什么：
- 公司业务 Summary（以后任何文件夹打开都能了解公司）
- 语言风格偏好
- 飞书 MCP 的连接路径（跑通一次就记下来）

项目级存什么：
- 当前产品的进展到哪里了
- 历史数据库在哪里读取
- 只属于这个项目的业务背景

写入方法：提示词可以很粗放，比如「请存档到这个项目下哟，我明天开机就要继续今天的进度哦」

## 飞书MCP机器人发消息提示没有权限怎么办？

原因：机器人没有被添加到目标群聊，或没有开启消息权限。

解决方法：
1. 确认机器人已被添加到目标群聊
2. 确认已开启 `im:message:send_as_bot` 权限
3. 确认应用已发布最新版本

## 飞书MCP无法给用户发私聊消息怎么解决？

原因：用户没有与机器人建立过会话。

解决方法：
1. 让目标用户先在飞书中搜索并打开你的机器人
2. 用户需要先给机器人发送任意一条消息，建立会话
3. 之后机器人才能主动给该用户发消息

## 飞书MCP找不到用户的 open_id 怎么查询？

原因：open_id 需要通过手机号或邮箱查询获取。

解决方法：在 Claude Code 中说：
- 「查询手机号 13800138000 对应的飞书用户信息」
- 或「查询邮箱 zhangsan@company.com 对应的飞书用户信息」

## 写提示词是不是要设计好流程、框架和模板？

不需要。提示词的本质就是用自然语言和 AI 沟通。

很多同学刚接触 AI 会觉得写提示词需要技术含量，需要提前设计好流程、拆 agent、准备模板框架。其实那套逻辑是做大型 AI 产品开发时才用得上的方法，不是日常工作里写提示词的标准动作。

举个例子：做公司项目管理看板，先让 AI 读几个 MD 文档，告诉它监控核心业务模块的关键节点和指标，讨论完了，最后给 AI 发一句「帮我做 HTML 页面」——这才是唯一的提示词。整个过程根本没有提示词框架，完全就是聊天。

与其研究怎么写提示词，不如先把自己要什么想清楚。想清楚了，AI 自然能帮你做出来。

## Claude 网页版 Memory 两个选项，开了能自动记住沟通背景吗？有缺点吗？

建议开启！

两个选项逻辑：
- Search and reference chats：你问问题的时候，它会去翻你的历史对话，找相关背景带进来
- Generate memory from chat history：它会从你的历史聊天里自动提炼关键信息存起来，下次直接用

一般来说不用担心记忆串联乱掉这种问题，偶尔可能提取得不太准，但影响不大。

最稳妥的还是靠自己的 CLAUDE.md、MD 文档、飞书多维表格存储来维护上下文，那个是你自己掌控的，不会被平台改来改去影响到。

## 想用 AI 提升工作效率，不知道第一步从哪里开始？

从一次随手的对话开始就够了。不管是用 Claude 网页版还是 Claude Code，第一步都一样——把你今天遇到的一个具体工作场景告诉 AI，跟它聊，把结果沉淀成一个 MD 文档。这就是起点。

举个真实例子：招聘时，先把公司招聘要求、简历筛选标准整理成 MD，告诉 Claude 帮你筛选简历、排序。这就是第一步。后续再沉淀更多场景，甚至搭看板，都是从一次随手对话展开的。

一句话：每天在做什么事，随手跟 AI 聊一个场景，沉淀成一个 MD，后面慢慢就有了。

## 用 AI 处理简历/做自动化工作流，简历数据从哪里来？不打通招聘平台接口就没法做吗？

不一定要打通接口，分两步来做：

第一步：先不管接口，把流程跑通。
简历先放多维表格或者本地文件夹，让 AI 读取、分析、输出结果。没有任何 MCP 或 API，先把基础方法论训练好，流程跑起来提效。

第二步：再研究能不能打通数据源。
有些平台有官方 API，有些没有，没有的话可以研究爬虫（让 AI 自己写）、人工录屏投喂、各种变通。

优先级放在人工获取信息后如何提效，而不是「找接口」这件事上。

## 给 Claude Code 丢了很多内容，但它感觉不长记性、记不住，怎么办？

「不长记性」的根本原因通常是上下文窗口满了。所有 AI 工具都有上下文限制，当你一次性丢了大量文档，它的「视野」很快就填满了。这时候它会自动做一次 Compaction（压缩），把前面的内容归纳成摘要，细节就丢了。这是正常机制。

更好的方式：
1. 先给目标，再给内容 — 开工时告诉它「我现在想做 XXX」，别让它漫无目的地读
2. 分批推进 — 把大任务拆成小块，一块一块来，不要一次性全丢
3. 定期让它归纳 — 每做完一块，让它总结当前进展，相当于「存档」
4. 及时压缩再继续 — 阶段完成后用 `/compact` 或 `/clear` 清空，开始下一阶段

核心：目的导向 + 分批推进 + 及时归纳，才是跟 AI 高效协作的正确姿势。

## 明明没用多少 Claude，怎么就显示用量到上限了？

这个提示是 Cursor 的用量上限，不是 Claude 的。

这两个完全是不同的东西：
- Cursor 内置 AI（Cursor 右边/底部的聊天框）：这是 Cursor 自己的 Agent，免费版有次数限制，到了就会提示 You've hit your usage limit
- Claude Code（课程里用的）：是在终端里启动的，输入 `claude` 回车才进入，用的是你自己的 Claude 账号额度，跟 Cursor 完全独立

解决方法：不要在 Cursor 的聊天框里对话，打开终端，输入 `claude`，看到小人图标出现，才是真正进入 Claude Code。

## 怎么让 Claude Code 省 token / 省钱？

1. 选对底薪——不同任务用不同模型：
   - 日常分析 → Sonnet 4.6
   - 简单任务/写初稿 → GLM、Kimi、MiniMax、豆包等国产模型
   - 复杂编程/关键决策 → Claude Opus 或国产高配模型

2. 先讨论再动手——Plan Mode 先行：
   开发是最费钱的环节，文字讨论比执行操作便宜很多。开始编程前先进 Plan Mode，讨论清楚方案再让 AI 动手。

3. 成功路径记下来，减少反复报错：
   某个流程跑通了，立刻告诉 AI「把这次成功的路径记录下来」，下次就不用重蹈覆辙。

4. 善用 `/clear` 和 `/compact`：
   对话太长后前面内容也占 token，阶段任务完成后及时压缩或清空。

5. 自己说话要清晰：
   你想清楚、说清楚，AI 就能一次执行到位，不用来回修正。

## Claude 网页版账号被封了怎么办？

被封是很多同学都经历过的必经过程，再去买个新账号就好。

关于先用豆包/元宝整理好再给 Claude Code 的思路：不太建议这个路子。原因是一定要先用 Claude 这种 top 模型，感受它的能力边界上限，再退而求其次用国产模型，这个顺序对学习很重要。

另外 Claude Code 不好控制 token 成本，课程第一个月以网页版为主，网页版成本是可控的。所以按课程节奏走，优先用网页版，封号了再买一个。

## Claude 网页版每天额度不够用，能用国内的卡充值升级 Pro/Max 吗？

不建议用境内卡直接充值，风险很高，成品号用国内卡付款容易因账号信息不一致被封号。

额度不够用的稳妥方案：
1. 多买几个账号轮着用 — 成品号直接买，几个账号错峰使用，比较稳
2. 升级到 Max 版本 — 大概 800 块左右，额度比 Pro 充足很多
3. Claude 搭配 GPT 一起用 — 简单任务交给 GPT，复杂的给 Claude，Claude 的额度能省不少
4. 后续把 Claude Code 跑起来 — Code + 网页版配合，完全够用了

## Claude 网页版只能上传 14 个文档，多文档怎么处理？

方法 1：分批处理
每次上传十几份资料，让它提炼重点或需要沉淀的信息，保存成 Markdown 文件；再上传下一批，继续沉淀。这样一批批处理下来，最后汇总就好。

方法 2：用 Claude Code
它可以直接读取本地某个文件夹，自动分批读取，对文件数量的限制更宽松。但压缩过程会丢失部分信息，需要定期让它保存阶段性成果，再继续读取——中间需要你人工干预一下。

方法 3：借助第三方模型
选择上下文更长、成本更低的模型（比如 Gemini 或其他国内模型），先由它们完成大量信息萃取并生成 Markdown，再将结果导入 Claude 进行后续工作。要注意其他模型的提炼能力没有那么强，特别重要的信息可能不太适合这么做。

## 怎么快速学会 Claude Code？感觉没掌握/运用不熟练怎么突破？

感觉没掌握是很正常的，CC 就是这样的工具，用得越多越顺手，光看录播找不到感觉的。

几个最有效的突破方式：
1. 提交作业、在直播间被点评：把真实业务场景做成课题提交，老师在直播间演示，比自己摸索快 10 倍
2. 在群里晒案例、看别人的：看同学们怎么用的，特别是跟自己业务类似的场景，直接参考改改就能用
3. 多参加直播答疑：有问题当场提，老师实操演示，比自己卡 2 小时效率高很多
4. 找宝姐要业务场景使用经验：告诉宝姐你具体在哪个场景卡住了，她能给你讲实际怎么用的

核心一句话：不要追求先学会再用，就是带着真实任务去用，用着用着就会了。

## 77 篇文章如何一次性喂给 AI？大量文档怎么处理？

直接把 77 篇全丢进去是不行的，会把 AI 的上下文窗口撑爆。

几个更好的方式：
1. 放本地文件夹批量读取：把所有文章放同一个文件夹，告诉 Claude Code 这个文件夹路径，让它分批按需读取，不是一次性全丢
2. 先用长文档 AI 提取再分析：用 NotebookLM、元宝这类支持超长上下文的工具先读完所有文章、提取关键信息，再把提炼后的内容给 Claude 分析
3. 先问自己：真的需要全读吗？大部分时候你只需要某个结论或某类观点，先把目标想清楚，针对性取几篇最相关的，效果往往比全扔更好
4. 逐步积累，每次沉淀一点：每批分析完，让 AI 输出一个小结文档，下次在这个基础上继续

## 数据看板怎么搭建？

核心在于先建表格，再建看板。虽然部分场景可跳过表格直接建看板，但如果看板与数据相关，底层数据结构越稳定，看板就越稳定。

在 Claude Code 中说「请你帮我生成 HTML 看板」，如需交互功能或特定风格（如商务风、黑底等），可一并说明。详细讲解见课程教程。
