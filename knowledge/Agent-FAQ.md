# Agent FAQ

## 最近很火的「多智能体协作」是什么？Claude Code 里怎么用？

大家说的「多智能体协作」（Agent Teams），本质就是让多个 AI 同时干活，而不是一个 AI 串行排队。

Claude Code 里有两个层面：

**自动层面** — Claude 内置了 Agent 工具，遇到复杂任务时会自动派多个「子代理」并行工作。比如同时搜索多个文件、同时跑多个分析。这个不需要你配置，Claude 自己判断什么时候该并行。

**手动层面** — 你可以配置 Claude Code 让它更聪明地干活：

1. **CLAUDE.md 定义项目规则**（最基础）— 告诉 Claude 这个项目是什么、怎么干
2. **Skills 封装常用工作流**（推荐）— 输入 `/命令名` 就执行一整套操作流程
3. **开多个 Claude Code 窗口**（进阶）— 每个窗口处理不同任务，各自独立

大多数同学用方式 1+2 就够了。方式 3 适合同时处理多个不相关的任务。

## MCP 和 Skills 有什么区别？

两个不同的概念，很容易搞混。

**MCP（Model Context Protocol）** 是连接外部工具的协议。比如连接飞书、连接数据库、连接搜索引擎。相当于给 Claude 接上"手和脚"，让它能操作外部系统。一般配 3-4 个常用工具就够，不需要装很多。

**Skills 是封装好的任务流程**。比如输入 `/批改作业` 就自动走"读表格→评分→写反馈"的完整流程。相当于给 Claude 写好了"操作手册"，一个命令触发一整套动作。

简单说：

- MCP = 给 Claude 装工具（连飞书、连搜索）
- Skills = 给 Claude 写流程（一键执行一整套操作）

大多数同学缺的是 Skills，不是 MCP。工具装好了但不会用，不如把常用流程沉淀成 Skills。

## CLAUDE.md 文件应该怎么写？是不是要把所有方法论知识都写进去？

CLAUDE.md 不是用来当知识库的，是用来告诉 AI「这是啥项目、需要啥就去哪读」。

更省力的思路：

1. CLAUDE.md 只写项目定位 + 调取规则（200行以内）
2. 方法论、话术库这些拆成单独文件存
3. AI 需要时自己去读，不用每次全加载
4. 让 AI 在工作中自己总结经验沉淀下来

说白了：你现在是在「手写百科全书」，其实应该让 AI「按需查字典 + 自己做笔记」。

好处：

- 多个项目复用同一套知识库
- 改话术一次，所有项目自动生效
- 每次不用加载大量内容，省 token 省钱
- 每个项目有自己的专属记忆
