# 数据安全 FAQ

## 财务数据有保密性要求，怎么用 AI 比较安全？

通过 API 调用的数据不会用于模型训练。

实操建议：脱敏 + 场景分层：
- 方法论/框架/模板放心用 AI
- 具体敏感数字手动填入

一句话：用 AI 做脑力活（框架、逻辑、模板），敏感数据自己填。

## 用 AI 写小红书内容总是违规/写不好？

先教 AI 规则，再让它放大：
1. 整理你的违规清单
2. 获取行业违规规则（问 AI + 搜平台）
3. 让 AI 建 Checklist 每次自检
4. 先手动打样几篇，教 AI 理解你的风格
5. 打通素材来源

一句话：先教规则，再放大，不要从零生成。

## 用第三方 AI 工具/平台，数据安全吗？会不会被后台看到或泄露？

没有百分之百绝对安全的工具。不管用什么产品，国内还是国外，底层都调用了某家厂商的模型。只要数据经过模型，就一定会留下痕迹。

相对原则：厂商越大、有公开声明，相对会好一点；厂商越小，风险相对更高。GPT 的 Teams 账号版本和 Claude 的 API 都有「不会把数据用于训练」的公开声明，但也无法做到百分之百安全。

整体上，需要在便利性和安全性之间自己做平衡，没有完美的答案。

## 真实业务数据有保密要求，脱敏后给 AI 分析偏差大，怎么处理？

方法一（主推）：公式模板法
1. 把你的表格结构描述给 AI（字段名、分析目标），不发真实数据
2. AI 生成带公式的 Excel 模板
3. 你在本地把真实数据填进去，Excel 自动算出结果
4. 把结果数字发给 AI，让它帮你写分析结论

优点：人人都能用，不需要技术背景，数据全程不离开你的电脑。

方法二（次推）：本地脚本运行法
让 AI 写 Python 分析脚本，你在自己电脑上运行，数据完全不上传。

运行逻辑：你的本地 Excel → 脚本读取 → Python 在你电脑上计算 → 输出结果。整个过程在电脑内部完成，不联网，不上传。

关键区别——数据会不会上传，取决于你怎么运行：
- 自己在终端运行脚本：完全不上传，100% 本地
- 让 Claude Code 帮你运行：数据会经过 API

安全做法：让 Claude 写脚本 → 自己在终端运行 → 只把结果发给 Claude 解读。

方法三（补充）：数据缩放法
把所有数据统一除以一个系数，发给 AI 分析，拿到结论后再换算回来。
- 适用：分析内部趋势、比例关系、同比环比
- 不适用：需要和行业对比、判断公司体量规模的分析

一句话总结：让 AI 帮你「想方法、写工具」，但数据的实际处理在你自己电脑上完成。
